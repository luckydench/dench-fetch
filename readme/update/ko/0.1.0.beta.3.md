## 0.1.0.beta.3 업데이트 (26.06.06)


### DenchURL
`DenchURL` 타입을 사용하실 수 있습니다. `DenchURL` 은 다음과 같이 정의되어 있습니다.

```tsx
export type DenchHTTPURL = 
`http://${string}` 
| `https://${string}` 
| `file://${string}` 
| `ftp://${string}` 
| `ws://${string}` 
| `wss://${string}`
```
이러한 타입을 만든 이유는 url 타이핑에 대한 실수를 줄이기 위해서입니다.

또한 이러한 타입을 dench 메서드에도 적용시켰습니다.

``` ts

export function dench<T>(baseURL: DenchHTTPURL, label? :string) : DenchInterface
```

**다만 특정 개발자는 baseURL을 자유로운 양식으로 타이핑하고 싶을 수 있다 생각하기 때문에, dench 메서드를 구체적으로 여러 갈래로 나눌 계획도 있습니다**

### auth 함수 업데이트 및 DenchAuthType

auth 함수 및 `DenchAuthType`이 업데이트 되었습니다. 이제 auth 함수는 다음과 같은 시그니처를 갖습니다.

```ts
    auth: (token: string, type? : DenchAuthType) => R,

    //구버전
    auth: (token: string) => R,
```

이전 버전에서는 auth 를 Bearer 방식만 인증 할 수 있는 형태였으나 **이제 DenchAuthType을 명시하면 원하는 방식으로 토큰 인증이 가능해집니다.** 지원되는 인증 방식은 아래와 같습니다.

``` ts
export enum DenchAuthType{
    ETC= 'ETC',
    BASIC = 'Basic',
    BEARER = 'Bearer',
    DIGEST = 'Digest',
    NEGOTIATE = 'Negotiate',
    AWS4_HMAC_SHA256 = 'AWS4-HMAC-SHA256',
    API_KEY = 'ApiKey',
    TOKEN = 'Token',
    HMAC = 'HMAC',
}
```
참고로 type을 명시하지 않을 시 기본 값은 **DenchAuthType.Bearer** 입니다.

만약 이 외의 인증 방식을 원하면 `DenchAuthType.ETC` 를 사용한 후 `token`을 **full string** 전부 입력하면 됩니다.
