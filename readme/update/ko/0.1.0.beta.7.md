## 0.1.0.beta.7 업데이트 (26.06.18)

### params 추가

params를 통해 url 파라미터를 설정할 수 있습니다.

```ts
params : (params : DenchURLSearchParams) => R

type DenchURLSearchParams = string | string[][] | Record<string, string> | URLSearchParams | undefined
```

#### 예제
```ts
const res = dench("http://example.com/").api("test").params("?a=apple&b=banana").toJson();

const res2 = dench("http://example.com/").api("test").params({ a : "apple", b: "banana"}).toJson();

```


### debug 추가

이제 debug 메서드를 통해 console 코드를 찍지 않고도 config가 어떻게 만들어지는 지 확인할 수 있습니다.

#### 예제
```ts
const res = dench("http://example.com/").api("test").params("?a=apple&b=banana").debug().toJson();

```

출력 형식은 다음과 같습니다.
```ts
`[${config.options.method}] ${config.label ? config.label : ""} Current Config:`, config
```

주의할 점 : debug는 그 시점에서의 빌더 체인의 options를 출력해줍니다. 만약 최종 설정 결과물을 확인하고 싶다면 toJson, toResponse등과 같은 dench runner 함수를 사용하기 직전에 사용하세요. 예를 들어


```ts
const res = dench("http://example.com/").api("test").params("?a=apple&b=banana").debug().auth("token", DenchAuthType.BEARER).toJson();
```

이렇게 debug를 사용하면 auth 에 관한 빌더 정보는 보이지 않습니다. 

만약 config가 만들어지는 과정을 확인하기 의도한 경우가 아니라면 최종 결과물을 확인하기 위해선


```ts
const res = dench("http://example.com/").api("test").params("?a=apple&b=banana").auth("token", DenchAuthType.BEARER).debug().toJson();
```
로 debug를 빌더 설정의 맨 끝에 사용하도록 하세요
