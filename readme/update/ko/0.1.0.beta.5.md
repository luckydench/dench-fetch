## 0.1.0.beta.5 업데이트 (26.06.07)


### api() 추가

이제 api를 나중에 설정할 수 있습니다.


#### 함수 시그니처
```ts
    /**
     * api URL을 설정합니다.
     */
    api: <P = T>(api: string) => R,
```

함수 시그니처를 봐도 알 수 있듯 `dench<T>(api)` 의 api 타입을 중도에 변경하거나, T타입을 P타입으로 변경할 수 있습니다.

---
### dench 생성 함수 시그니처 변경

- 이제 더이상 dench에 api를 쓸 필요가 없어졌습니다.
- post, put 또한 dench 생성시 data를 넣지 않습니다.

#### 함수 시그니처
```ts
    export interface DenchInterface{
        baseURL : DenchHTTPURL,
        get : <T>(api?:string) => DenchGetBuilder<T>
        post : <T>(api?: string, data?: unknown) => DenchCreateBuilder<T>
        put : <T>(api?: string, data?: unknown) => DenchCreateBuilder<T>
        delete : <T>(api?: string) => DenchGetBuilder<T>
    }
```

---
### data 설정 선택 옵션으로 변경
- 이제 sendJson, sendFormData, sendBlob함수에 data를 넣을 수 있습니다

#### 함수 시그니처
```ts
    sendJson : (data? : unknown) => DenchCreateBuilder<T>,
    sendForm : (data? : FormData) => DenchCreateBuilder<T>,
    sendBlob : (data? : Blob) => DenchCreateBuilder<T>,
```
만약 이미 dench.put 또는 post를 생성할 때 data를 넣었다면 보내는 정보가 덮어쓰기가 되므로 주의해야 합니다.

---
### sendRaw, sendURLParams, DenchURLSearchParams 추가

- 이제 URLParmas를 보내는 sendURLParams와 그냥 원본 데이터 그대로 보내는 sendRaw를 사용할 수 있습니다.

```ts
    type DenchURLSearchParams = ConstructorParameters<typeof URLSearchParams>[0]

    /**
     * @header { "Content-Type" : "application/x-www-form-urlencoded" }
     */

    sendUrlEncoded? : (data?  : DenchURLSearchParams) => DenchCreateBuilder<T>

    /**
     * @header { "Content-Type" : "application/octet-stream" }
     */
    sendRaw? : (data? : BodyInit) => DenchCreateBuilder<T>
```

`DenchURLSearchParams`는 sendUrlEncoded의 매개 변수 인자를 엄격하게 다루기 위한 타입으로, URLSearchParams의 생성자 타입과 같습니다. 이는 아래와 동일합니다.
```ts
 type DenchURLSearchParams = string | string[][] | Record<string, string> | URLSearchParams | undefined
 ```

`sendRaw`는 개발자 본인이 직접 데이터를 가공하여 그대로 body에 집어넣고 싶을 때 사용할 수 있습니다.
이미 `Json.stringify()`, `FormData()`등의 작업을 거친 데이터를 넣게 된다면 sendRaw를 사용하면 됩니다.

### copy() 메서드

이제 `copy()`를 통해 builder를 복사할 수 있습니다.

해당 기능을 추가한 이유는 builder의 재사용성을 높이기 위함입니다. 아래 예제를 보면 더 쉽게 이해 할 수 있습니다.

```ts
const denchInstance = dench('https://jsonplaceholder.typicode.com', "testInstance");


const sampleBuilder = denchInstance.post()
                .abort(new AbortController())
                .timeout(5000)
                .error((error) => console.error('Error occurred:', error))

const builder1 = sampleBuilder.copy()
.sendJson({
    userId : 1,
    title : 'delectus aut autem',
    completed : false
})
.api('/todos?userId=1');

const builder2 = sampleBuilder.copy()
.sendForm(new FormData())
.api('/todos?userId=1');
```
위와 같이 `error`, `timeout` 등 공통적으로 들어가는 설정을 재사용하기 위해선 구조 분해로 복사를 하거나 아예 새로운 builder를 만들어야 했으나, 이제 `copy`를 사용하면 기존의 빌더 설정을 복사하여 위와 같이 재사용할 수 있습니다.
