## 0.1.0.beta.7 Update (26.06.09)

### Added `params()`

You can now configure URL query parameters through the builder chain with `params()`.

```ts
params: (params: DenchURLSearchParams) => R

type DenchURLSearchParams =
  | string
  | string[][]
  | Record<string, string>
  | URLSearchParams
  | undefined
```

#### Example

```ts
const res = dench("http://example.com/")
  .api("test")
  .params("?a=apple&b=banana")
  .toJson();

const res2 = dench("http://example.com/")
  .api("test")
  .params({ a: "apple", b: "banana" })
  .toJson();
```

---

### Added `debug()`

You can now inspect the current builder config with `debug()` without adding your own console code.

#### Example

```ts
const res = dench("http://example.com/")
  .api("test")
  .params("?a=apple&b=banana")
  .debug()
  .toJson();
```

The output format is:

```ts
`[${config.options.method}] ${config.label ? config.label : ""} Current Config:`, config
```

`debug()` prints the builder options at the point where it is called in the chain.
If you want to inspect the final request config, call it immediately before a Dench runner method such as `toJson()` or `toResponse()`.

For example, this call does not include the later auth configuration in the debug output:

```ts
const res = dench("http://example.com/")
  .api("test")
  .params("?a=apple&b=banana")
  .debug()
  .auth("token", DenchAuthType.BEARER)
  .toJson();
```

To inspect the final config, place `debug()` at the end of the builder setup:

```ts
const res = dench("http://example.com/")
  .api("test")
  .params("?a=apple&b=banana")
  .auth("token", DenchAuthType.BEARER)
  .debug()
  .toJson();
```
